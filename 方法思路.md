# 面向长尾解释生成的 Token Graph 方法思路

## 1. 目标与问题定义

本方法面向基于用户历史评论生成解释（explanation）的任务。给定用户的历史评论、当前待解释商品及其可见元数据，模型需要生成一段既与用户偏好一致、又能描述当前商品特征的文本解释。

普通监督微调（SFT）容易被训练集中高频的评价词主导，例如“great”“good”“quality”。这类词即使在测试集上频繁出现，也未必构成区分商品或用户偏好的有效证据；相对地，低频但具体的内容词更难被预测。与此同时，按用户历史词频截断的 token graph 往往会把低频词在进入 selector 前就丢弃。方法的核心目标是：**提升低频内容 token 构成的解释的可预测性，同时不把罕见词错误地等同于证据词。**

为此，整体设计由三部分组成：

1. 使用训练集统计的 tail-weighted SFT，使生成器更重视被预测错的低频内容 token；
2. 在用户历史 token graph 上使用监督式 selector，学习哪些历史 token 会实际出现在当前目标 explanation 中；
3. 用“长尾—商品相关—稳定偏好”的分层规则保留至多 512 个图节点，避免简单高频截断。

## 2. 记号与泄漏安全约束

对于一个样本，令目标 explanation 的 token 序列为 \(y=(y_1,\ldots,y_T)\)，用户历史中可用于构图的评论集合为 \(\mathcal H\)，当前商品为 \(i\)。

构图时必须排除：

- 当前样本本身；
- 用户对当前目标商品 \(i\) 的所有评论。

因此，图中信息只来自推理时可见的用户历史。当前商品的 title、description 等元数据可用于衡量节点与商品的相关性；真实 explanation、keyword_words 等标签侧信息不得参与节点选择。

训练、验证和测试阶段都只使用当前 fold 的训练 explanation 计算全局 token 频率。这样可以避免验证集和测试集标签泄漏到重加权、负例采样或图节点保留规则中。

## 3. 训练集长尾统计与权重

### 3.1 内容 token 与 document frequency

首先过滤 special token、标点、纯数字、停用/无意义短子词等非内容 token。对训练 explanation 中的内容 token，定义 document frequency：

\[
df(t)=\#\{y\in\mathcal D_{\rm train}\mid t\in y\}.
\]

这里按 explanation 文档计数，而不是按同一条 explanation 内的重复次数计数。令 \(d_{\rm ref}\) 为所有训练 explanation 的内容 token 位置所对应 \(df(t)\) 的第 75 百分位数。换言之，高频 token 因为在更多位置出现，对该参考值的影响也更大。

对内容 token 使用如下长尾权重：

\[
r(t)=\left(\frac{df(t)+1}{d_{\rm ref}+1}\right)^{-\alpha},
\qquad
w_{\rm tail}(t)=\operatorname{clip}\bigl(r(t),w_{\min},w_{\max}\bigr).
\]

固定 \(\alpha=0.5\)，默认 \(w_{\min}=0.5\)、\(w_{\max}=2.0\)。只在验证集上搜索：

\[
w_{\min}\in\{0.5,0.75\},\qquad
w_{\max}\in\{1.5,2.0,2.5\}.
\]

该设计的含义是：相对稀有的内容 token 权重大于 1，相对流行的内容 token 权重可低于 1；权重截断防止极罕见 token 产生不稳定的大梯度。训练集中从未出现的 token 在验证/测试时权重固定为 1，而不是被任意放大。非内容 token 的权重也固定为 1。

定义长尾候选的频率阈值为：

\[
df(t)\leq \max\left(5,\left\lceil0.001\,|\mathcal D_{\rm train}|\right\rceil\right).
\]

该阈值只用于图节点分层保留；损失中的 \(w_{\rm tail}\) 则由连续的频率函数给出。

## 4. Token Graph 构建与节点编码

### 4.1 图结构

从泄漏安全的用户历史 \(\mathcal H\) 抽取合法内容 token 作为候选节点。对同一条历史评论中先出现的 token 与后出现的 token 建立有向边；同一商品的多条历史评论也可合并后补充共现边。由此得到用户级 token graph：

\[
G=(V,E).
\]

每个节点仍记录局部 count、局部 document frequency 和入/出度，以支持缓存、分析和 selector 最终打分；但它们不再拼接到 GNN 的节点输入。

### 4.2 分层保留 512 个节点

不再只取用户历史中 count 最高的 512 个 token。对于去重后的合法候选节点，按以下顺序保留，每一步都跳过已被选中的节点：

| 节点组 | 配额 | 排序原则 | 目的 |
| --- | ---: | --- | --- |
| 全局长尾节点 | 256 | 低全局 \(df\) 优先；与当前商品 title/description 的 token 重合优先；再按用户局部频率排序 | 确保可能有区分力的低频词能进入图 |
| 商品相关节点 | 128 | 与当前商品可见元数据 token 重合优先；再按用户局部频率排序 | 提升当前商品属性的覆盖 |
| 稳定偏好节点 | 128 | 用户历史局部 document frequency、再 count 由高到低 | 保留用户反复表达的偏好 |

若某一组不足配额，或三组总数不足 512，则以“商品相关优先、用户局部频率其次”的确定性顺序回填。所有排序均使用确定性 tie-break，保证缓存可复现。

该步骤只改变**哪些节点进入图**，不使用真实目标 explanation。因此，后续的 tail coverage、selector recall 与生成质量可以被分别诊断。

### 4.3 删除五维统计节点特征

节点的初始表示只使用冻结的 Qwen3-Embedding（不可用时使用冻结的语言模型 embedding）得到的 token 向量 \(e_v\)。不再将以下五维统计量拼接到 GNN 输入：

\[
[\log(1+\mathrm{count}),\ \log(1+\mathrm{df}),\ \mathrm{idf},\ \mathrm{norm\ in\ degree},\ \mathrm{norm\ out\ degree}].
\]

因此，图编码为：

\[
h_v=\operatorname{GraphSAGE}(\operatorname{MLP}(e_v),E).
\]

两层 GraphSAGE 继续聚合图拓扑信息，但节点表示不再直接把“越高频”或“度越大”当成先验。selector 的末端可保留独立的三维局部频率旁路

\[
[\log(1+\mathrm{count}),\ \log(1+\mathrm{df}),\ \mathrm{count}/\max\mathrm{count}],
\]

它只参与最终 evidence 打分，而不污染 GNN 的语义/结构编码；其潜在高频偏置由后续 tail 加权的 selector 监督纠正。

## 5. Tail-weighted SFT

令 \(m_{b,j}\) 是 batch 中第 \(b\) 个样本第 \(j\) 个目标位置的有效 mask，\(\operatorname{NLL}_{b,j}\) 是常规语言建模负对数似然。定义位置权重：

\[
a_{b,j}=
\begin{cases}
1, & y_{b,j}\text{ 为功能词、标点或 special token},\\
w_{\rm tail}(y_{b,j}), & y_{b,j}\text{ 为内容 token}.
\end{cases}
\]

生成器的监督微调损失为：

\[
L_{\mathrm{SFT}}^{\mathrm{tail}}
=
\frac{\sum_{b,j}m_{b,j}a_{b,j}\operatorname{NLL}_{b,j}}
{\sum_{b,j}m_{b,j}a_{b,j}}.
\]

该损失提高低频内容 token 预测错误的梯度贡献，同时仍保留语言流畅性所需的功能词、标点和常见内容词。现有特征损失 \(L_{\rm feat}\) 保持原定义，不额外使用 tail 权重。

## 6. 监督式 Evidence Selector

### 6.1 正例定义

selector 对图中每个节点 \(v\) 输出原始效用分数 \(z_v\)，其概率为：

\[
p_v=\sigma(z_v).
\]

正例集合为图节点与真实目标 explanation 的内容 token 的交集：

\[
P=\{v\in V\mid \operatorname{id}(v)\text{ 出现在当前目标 explanation 中}\}.
\]

这意味着 selector 学习的是“给定用户历史和当前商品，哪些历史 token 能预测本次真实 explanation”，而不是把所有稀有 token 一律视为证据。若 \(P\) 为空，当前图没有可用的 grounded supervision，令该样本的 selector loss 为 0；不能把整张图错误标为负例。

### 6.2 负例采样

对每个正例，从 \(V\setminus P\) 中构造至多 8 个负例：

1. 4 个 hard negatives：当前 selector 分数最高、但未命中目标 explanation 的节点；
2. 2 个 popularity-matched negatives：与正例处于同一全局频率桶的随机节点；
3. 2 个 popular negatives：图中全局 \(df\) 最高的节点。

频率桶定义为：

\[
b(t)=\lfloor\log_2(df(t)+1)\rfloor.
\]

若同频桶不足两个节点，则由距离最近的频率桶回填；候选节点过少时再按 hard/popular/其余候选的确定顺序补足。硬负例的“hard”只用于选择节点，选择时对分数 detach；被选中的负例仍通过 BCE 向 selector 回传梯度。

同一负例可服务于不同正例，因而在一个样本的损失中可按其被采样的次数计入；这实现了“每个正例配 8 个对照负例”的监督密度。采样随机性仅来自同频桶的随机节点，实验时应固定随机种子。

### 6.3 单一的加权 BCE 损失

使用标准的加权二元交叉熵形式，但 \(V\) 仅表示当前样本被采样到的 \(P\cup N\)，而不是整张最多 512 节点的图：

\[
L_{\mathrm{selector}}
=-\frac{1}{Z}\sum_{v\in P\cup N}
\left[
w_v^+y_v\log p_v+
w_v^-(1-y_v)\log(1-p_v)
\right].
\]

其中 \(y_v=1\) 表示正例，正负权重设置为：

\[
w_v^+=\frac{|N|}{|P|}\cdot w_{\rm tail}(v),\quad v\in P;
\qquad
w_v^-=1,\quad v\in N,
\]

\[
Z=\sum_{v\in P}w_v^++\sum_{v\in N}w_v^-.
\]

\(|N|/|P|\) 先抵消“每个正例约有 8 个负例”造成的类别不平衡；在此基础上，长尾正例再通过 \(w_{\rm tail}\) 获得更强监督。与“正例项、负例项分别归一化后再各占一半”的写法相比，此形式更通用：正负比例和长尾强度都显式体现在 \(w_v^+\)、\(w_v^-\) 中。

## 7. 总训练目标与 Evidence 使用方式

最终优化目标为：

\[
L=L_{\mathrm{SFT}}^{\mathrm{tail}}
+\lambda_{\mathrm{selector}}L_{\mathrm{selector}}
+\lambda_{\mathrm{feat}}L_{\mathrm{feat}}.
\]

初始设置为 \(\lambda_{\mathrm{selector}}=0.1\)，只在验证集上搜索：

\[
\lambda_{\mathrm{selector}}\in\{0.05,0.1,0.2\}.
\]

不使用以下目标或机制：

- 不使用 pairwise ranking loss；
- 不使用 unlikelihood loss \(L_{\rm UL}\)，也不再需要 UL 的负 token 选择和权重；
- 不使用连续/可微的 evidence bonus、soft selection、温度参数或 tail-scaled logit bonus。

训练时 selector 的梯度完全由 \(L_{\mathrm{selector}}\) 提供。推理时仍可保留原有的 hard Top-M evidence token 机制：选择分数最高的 \(M=5\) 个图节点，并对其对应生成 token 增加固定的非可微 logit bonus。默认 bonus 为 0.1；若希望完全关闭这一遗留的解码偏置，只需设置 `evidence_bonus=0`，其余训练方案不变。

## 8. 训练与推理流程

### 8.1 训练流程

1. 仅从当前 fold 的训练 explanation 计算 \(df(t)\)、\(d_{\rm ref}\)、\(w_{\rm tail}(t)\) 和长尾阈值；
2. 对每个样本，以泄漏安全历史和当前商品可见元数据构建分层 token graph；
3. 用冻结 token embedding 和 GraphSAGE 得到节点表示，并与当前商品表示一起计算 selector 分数；
4. 从目标 explanation 与图节点交集构造正例，采样三类负例，计算 \(L_{\rm selector}\)；
5. 用 target token 的 \(w_{\rm tail}\) 计算 \(L_{\mathrm{SFT}}^{\mathrm{tail}}\)，并与 selector/特征损失联合反向传播；
6. 将训练集统计版本、权重参数和 256/128/128 配额写入图缓存 fingerprint。任一项变化均应重建缓存。

### 8.2 推理流程

推理时不访问真实 explanation。模型以用户的泄漏安全历史、当前商品元数据和训练期固定的 tail 统计构建图，计算 selector 分数并取 hard Top-5 evidence token。生成器按常规自回归解码；若启用静态 bonus，则只对这五个 token 施加固定 bonus。该过程不会让 selector 通过 top-k 操作反向传播梯度，也不会在推理期使用标签侧信息。

## 9. 消融与诊断方案

建议按以下顺序验证每个组件的作用：

1. 原始模型；
2. 原始模型 + tail-weighted SFT；
3. 上一步 + 监督式 \(L_{\mathrm{selector}}\)；
4. 上一步 + 分层 512 节点；
5. 在最佳组合上搜索 \(w_{\min}\)、\(w_{\max}\) 与 \(\lambda_{\mathrm{selector}}\)。

除总体 BLEU、ROUGE、FMR、Distinct 等指标外，应单独报告以下诊断指标：

| 指标 | 含义 | 可定位的问题 |
| --- | --- | --- |
| TailGraphCoverage@512 | 真实 tail 内容 token 中有多少进入了图 | 长尾 token 是否在截断阶段已丢失 |
| TailEvidenceRecall@5 | 图内真实 tail 正例中有多少进入 selector Top-5 | 图已覆盖但 selector 是否选错 |
| Tail-token Precision / Recall | 生成解释对真实 tail 内容 token 的准确率与召回率 | 生成器能否真正预测长尾内容 |
| Tail-FMR | 围绕长尾内容的特征匹配比 | 长尾词是否具有可解释的属性一致性 |
| 总体 FMR、BLEU、ROUGE、Distinct | 整体匹配、文本相似度和多样性 | 长尾改进是否牺牲总体质量 |

只有同时观察“图覆盖—selector 选择—生成结果”三层指标，才能区分以下失败原因：低频 token 根本没有进入图、进入图但 selector 未选中、selector 已选中但生成器仍偏向流行 token。

## 10. 方法边界与待验证假设

本方法学习的是“最能预测真实 explanation 的图内内容 token”，并不自动等价于严格因果意义上的真实证据。若研究目标要求因果证据，还需要人工 evidence 标注、用户点击/购买反馈或额外的反事实信号。

本方法的主要经验性假设是：训练 explanations 的高频分布会掩盖低频内容词，且用户历史中存在对当前商品 explanation 有帮助的可见证据。以下结论必须由实验支持，而不应在实验前宣称为既成事实：

| 待验证主张 | 所需证据 |
| --- | --- |
| tail-weighted SFT 提升长尾内容预测 | Tail-token Precision/Recall、Tail-FMR 及显著性检验 |
| selector 能抑制“只选高频词”的偏置 | TailEvidenceRecall@5 与各频率桶的 evidence 分布 |
| 分层 512 节点改善长尾可用性 | TailGraphCoverage@512 及与高频截断基线的比较 |
| 整体质量未被明显牺牲 | 总体 FMR、BLEU、ROUGE、Distinct 和效率开销 |
